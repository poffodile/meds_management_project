## 1. Folder rules (so code is structured, never scattered)

**One rule: every file has exactly one correct home.** Before creating a file, find its home below.

```
frontend/                         ← shared, reusable code (used by many pages). Import via '@frontend/...'
├── components/                   ← generic UI blocks: StatCard, DataTable, PageHeader, FormModal, StatusBadge…
├── Layouts/                      ← page shells: AppShell.jsx (sidebar + top bar)
├── lib/                          ← helpers, hooks, context: role.js, formatting, the api client
└── theme.js                      ← design tokens (colours, fonts, spacing) — the ONE place for the look

resources/js/
├── app.jsx                       ← the entry point (don't put UI here)
└── Pages/                        ← ONE file per screen, grouped by module. Inertia auto-loads these.
    └── Medication/Stock.jsx      ← mirrors the route /medication/stock

resources/views/app.blade.php     ← the HTML shell React mounts into
app/Http/Controllers/...          ← Laravel controllers return Inertia::render('Module/Page', $props)
```

**The rules:**
1. **A page** (one screen) → `resources/js/Pages/<Module>/<Name>.jsx`. Keep it **thin**: receive data, arrange components.
2. **Anything reused by 2+ pages** → `frontend/components/`. Never copy-paste a component between pages.
3. **A shell/layout** → `frontend/Layouts/`.
4. **A helper, hook, or context** → `frontend/lib/`.
5. **Any colour/font/spacing** → only in `frontend/theme.js`. No hard-coded colours in pages.
6. **Names:** Components & Pages = `PascalCase.jsx`. Helpers = `camelCase.js`.
7. **Imports:** shared code is always `@frontend/...` (never long `../../../` paths).

## 2. The pieces of M0 (with how to test each)

### Step 1 — Folder structure & conventions  ✅
This document. Everyone (and every future file) follows section 1.

### Step 2 — Design tokens / theme  🔄 (`frontend/theme.js`)
The single place that controls colours, fonts, spacing.
- **Test:** change the primary colour in `theme.js`, refresh → the whole app's accent colour changes.

### Step 3 — App shell (frame)  ✅ (`frontend/Layouts/AppShell.jsx`)
Sidebar + top bar + user menu, **role-aware** (Manager vs Carer), with a dev preview toggle.
- **Test:** see the manual checklist in section 3.

### Step 4 — Role & permissions plumbing  ✅ (`frontend/lib/role.js` + backend share)
Backend tells the frontend if you're `manager` or `carer`; pages use `useRole()` to show/hide actions.
- **Test:** toggle Manager/Carer → menu and the "Adjust stock"/"View only" control change.

### Step 5 — Shared component kit  🔄 (`frontend/components/`)
The reusable blocks every page needs. Build the core set now:
- ✅ `StatCard` — number tile.
- ✅ `PageHeader` — title + actions row (so every page's top looks the same).
- ✅ `DataTable` — search + sortable columns + pagination (used everywhere).
- ✅ `StatusBadge` — coloured status pill with one shared colour map.
- ⬜ `EmptyState` — the "no data yet" message. *(currently handled by DataTable's emptyMessage)*
- ✅ `FormModal` + `ConfirmDialog` — pop-up form and "are you sure?" dialog.
  - Wired live: the **Adjust stock** action (`frontend/features/medications/AdjustStockModal.jsx`) → posts to `medication.stock.react.adjust`; disposals trigger a `ConfirmDialog`.
- **Test:** each component has a tiny render test (section 4) and is used on the pilot page.

### Step 6 — Testing foundation  ✅
Test tools set up on both sides, with a working example of each:
- **Backend:** PHPUnit (existing). Run: `php artisan test`. Example: `tests/Feature/MedicationStockReactTest.php`.
- **Frontend:** Vitest + React Testing Library. Run: `npm run test`. Example: `frontend/components/StatusBadge.test.jsx`.
- Setup files: `vitest.config.js`, `frontend/test/setup.js`, `frontend/test/render.jsx`.

### Step 7 — Pilot page proves it all works  ✅ (`/medication/stock-react`)
The Medication Stock page uses the theme + shell + role + components together.

## 3. Manual tests — click through these (the "does it work?" checklist)
Log in first, open **http://127.0.0.1:8000/medication/stock-react**, then check:
1. ⬜ Page loads with **no red errors** in the browser console.
2. ⬜ **Shell shows**: left sidebar, top bar, "CareOS" brand, your initial in a circle (top-right).
3. ⬜ **Role badge** (top bar) matches your account (Manager portal / Carer view).
4. ⬜ **Preview = Manager** → sidebar shows the full menu; an **"Adjust stock"** button shows on the page.
5. ⬜ **Preview = Carer** → sidebar shows the short menu; the button becomes a **"View only"** badge.
6. ⬜ **Data is correct** → stat cards show numbers; Overview lists medications; Transactions lists history; matches the old `/medication/stock` page.
7. ⬜ **Tabs** switch between Overview and Recent Transactions.
8. ⬜ **Responsive** → make the window narrow; a ☰ burger appears and opens/closes the sidebar.
9. ⬜ **User menu** → click your initial → shows your name + Log out.
10. ⬜ **Active menu** → "Medication Stock" is highlighted; greyed items don't click.
